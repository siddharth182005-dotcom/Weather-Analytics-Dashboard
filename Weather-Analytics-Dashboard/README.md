# 🌤️ Weather Analytics Dashboard Pro

A production-ready **Power BI** analytics project with a dark glassmorphism UI, a 10-city Indian weather dataset spanning a full year, a full DAX measure library, and Power Query transformations. Built to look and behave like a premium Microsoft dashboard gallery entry.

> ⚠️ **Important note on the `.pbix` file**: A `.pbix` is a proprietary binary format that only Power BI Desktop can create and save — it cannot be generated by a script or text editor. This project therefore ships every *ingredient* of the `.pbix` (data, theme, DAX, Power Query, icons, layout spec) fully built, so that assembling the actual file in Power BI Desktop takes minutes, not hours. See **Installation** below for the exact steps.

## 3. Installation — Building the `.pbix`

1. **Install** Power BI Desktop (free, Microsoft Store or powerbi.microsoft.com).
2. **Open** a new blank report.
3. **Get Data → Text/CSV**, import each file in `/Data`: `City.csv`, `Calendar.csv`, `Weather.csv`, `Forecast.csv`, `AQI.csv`.
4. In **Power Query Editor**, open the **Advanced Editor** for each query and paste the matching block from `PowerQuery/Queries.txt` (it already handles null-value replacement, type correction, renaming, and includes merge/append examples). Rename the `AQI` query to `AirQuality` to match the DAX below.
5. **Close & Apply.**
6. Go to **Modeling → New Table**, create a measures table: `_Measures = ROW("_Measures", BLANK())`. Add each measure from `DAX/Measures.txt` (40+ measures) inside it.
7. **Model view**: build the star-schema relationships (Section 4 below).
8. **View → Themes → Browse for themes**, select `Theme/theme.json`.
9. Build the report pages using the **Dashboard Layout** spec in Section 6 and the mockup in `Screenshots/01_Overview.png` as a visual reference. Drop the SVG icons from `Assets/Icons/` in as image visuals next to each stat card.
10. **Save As → Weather Dashboard.pbix.**

Total build time following this guide: ~15–20 minutes.

## 4. Data Model (Star Schema)

```
                 ┌─────────────┐
                 │  Calendar   │
                 │  (Date PK)  │
                 └──────┬──────┘
                         │ 1:*
        ┌────────────────┼────────────────┐
        │                │                │
   ┌────▼─────┐    ┌─────▼─────┐   ┌──────▼──────┐
   │ Weather  │    │ Forecast  │   │ AirQuality  │
   │ (fact)   │    │ (fact)    │   │ (fact)      │
   └────┬─────┘    └─────┬─────┘   └──────┬──────┘
        │ *:1            │ *:1            │ *:1
        └────────────────┼────────────────┘
                         │
                    ┌────▼────┐
                    │  City   │
                    │ (CityID)│
                    └─────────┘
```

- **City** (dimension) — CityID, City Name, State, Lat/Long, Region, BaseTemp
- **Calendar** (dimension, marked as Date table) — Date, Day, Month, Quarter, Year, Season, IsWeekend
- **Weather** (fact) — daily actuals per city: Temperature, FeelsLike, Humidity, Wind, Pressure, Visibility, Cloud Cover, Precipitation, UV Index, Condition
- **Forecast** (fact) — 7-day-ahead forecast per city: Max/Min/Avg Temp, Rain Probability, Sunrise/Sunset
- **AirQuality** (fact) — daily AQI + PM2.5, PM10, CO, NO2, SO2, O3 per city

All fact tables relate to `City` (many-to-one on `CityID`) and to `Calendar` (many-to-one on `Date`), keeping every table on the "many" side — no bidirectional filtering needed, which keeps performance fast even across 3,650+ row fact tables.

## 5. Power Query

See `PowerQuery/Queries.txt` for the full scripts. Highlights:

- **Null handling**: missing humidity/wind readings replaced with sensible defaults; missing condition text defaulted to `"Clear"`.
- **Type correction**: every column explicitly typed (Int64, number, date, text, logical).
- **Renaming**: `City` → `City Name` to avoid ambiguity with the table name.
- **Merge example**: Weather ⨝ City to pull Region onto the fact table for fast tooltips.
- **Append example**: Weather (actuals) + Forecast combined into one continuous `WeatherTimeline` query for a single line chart spanning past and future.
- **Calculated columns**: `FeelsLike Diff`, `IsRainy`, `Temp Range`, `SeverityRank`.
- **Live-API variant**: an optional query that calls WeatherAPI.com directly via `Web.Contents`, if you'd rather wire the dashboard to live data than the bundled CSVs (see `Assets/sample_api_response.json` for the expected JSON shape).

## 6. Dashboard Layout

The layout follows a proven "weather app" reference style (icon-first forecast strip, gauge + colored-dot pollutant tiles, horizontal probability bars) rather than a generic grid of charts — it's the same structure taught in widely-used Power BI weather-dashboard tutorials, rebuilt here with this project's own data model, DAX, and dark glassmorphism theme.

**Left panel** (tall card, purple-to-dark gradient) — location pin + name, "Last Updated" timestamp, weather emoji/icon, large current temperature, condition text, and three horizontal city-selector chips (selected city highlighted with the blue→purple gradient pill).

**Below the left panel** — a 2×2 grid of Humidity, Wind Speed, Visibility, and Pressure cards (Feels Like, Cloud Cover, UV Index, and Precipitation are available as the same card component for a Forecast-page variant).

**Top strip** — five compact forecast chips, one per upcoming day, each with a condition icon, day name, and temperature; alternate days shown at reduced opacity to match the reference's "today + selected days" emphasis pattern.

**Center** — "Weather Forecast" line chart with floating temperature-value pills above each point instead of axis labels, smooth line, and day names along the bottom.

**Top right** — Sunrise and Sunset card with icon rows, plus Golden Hour and Day Length as secondary stats below a divider.

**Bottom center** — Air Quality Overview: a ring gauge (AQI value + Good/Moderate/Poor/Hazardous label at center) alongside six pollutant tiles (PM10, PM2.5, O3, NO2, SO2, CO), each with a small colored status dot bound to the matching `*_Status Color` DAX measure, plus a one-line air-quality summary caption below the gauge.

**Bottom right** — Chances of Rain: horizontal gradient bars per day with a percentage value at the end and a 0/50/100% axis strip beneath.

A full-resolution mockup of this exact layout is in `Screenshots/01_Overview.png`.

**Additional pages:** Forecast (full 7-day detail + hourly breakdown), Air Quality (city-by-city AQI matrix + trend), Historical Trends (365-day line/area charts, rolling averages), City Comparison (small multiples / matrix across all 10 cities), Weather Alerts (conditional table driven by threshold measures, e.g. AQI > 200 or Rain Probability > 70%).

## 7. DAX

`DAX/Measures.txt` contains 40+ measures grouped into:

1. Core temperature (Current, Average, Max, Min, Feels Like, Heat Index, Temperature Difference)
2. Atmospheric (Humidity, Wind, Pressure, Visibility, Cloud Cover, UV Index, Precipitation)
3. Forecast (Forecast Average, Rain Probability, Sunrise, Sunset, Day Length, Golden Hour)
4. Trend/rolling (7-day Rolling Average, Trend %, Month-over-Month change)
5. Air quality (Average AQI, PM2.5, PM10, CO, NO2, SO2, O3, AQI Category)
6. **Dynamic KPI colors** — hex-string measures (`AQI Color`, `Temperature KPI Color`, `Rain Probability Color`, `Trend Arrow`) designed to be bound via Conditional Formatting → Font Color → Field Value
7. Composite score (`Weather Score`, 0–100, weighted blend of temp/humidity/AQI/rain/visibility)
8. Utility (`Last Updated`, unit-toggle-aware `Display Temperature` for the °C/°F switch)

## 8. Customization

- **Colors**: edit `Theme/theme.json` — `dataColors` array controls chart series; `background`/`foreground` control the base dark palette.
- **Add a city**: append a row to `City.csv` with a new `CityID`, then add matching rows to `Weather.csv` / `Forecast.csv` / `AQI.csv` (or point the live-API query at it instead).
- **Switch to live data**: swap the CSV-based queries for the WeatherAPI.com version at the bottom of `PowerQuery/Queries.txt`; requires a free API key from weatherapi.com stored as an `APIKey` parameter.
- **Unit toggle**: create a disconnected `Unit Parameter` table with values `{"C","F"}`, add a slicer, and use the `Display Temperature` / `Temp Unit Toggle Label` measures already included.

## 9. Deployment

- **Power BI Service**: Publish from Desktop (`Home → Publish`) to a workspace; schedule refresh if you've wired the live-API query (requires an on-premises data gateway if the CSVs stay local, or none if fully cloud-based via WeatherAPI.com).
- **Power BI Embedded**: the same `.pbix` can be embedded via `powerbi-client` in a custom web app once published to a Premium/Embedded capacity.
- **Sharing as a template**: File → Export → Power BI Template (`.pbit`) strips the data and keeps the model/report shell, useful for distributing the design without the sample dataset.
